# LLM Proxy Implementation Plan

**Goal:** Build a multi-worker Cloudflare Worker proxy that rotates exit IPs via deterministic fake IP generation, allowing a single PC to use multiple API keys across different upstream LLM providers.
**Testing Conventions:** Vitest with `globals: true`, Node environment. Tests in `tests/` mirroring `src/` structure. Pure unit tests for utilities, mock-based tests for router/proxy handlers (mock `fetch`, mock env bindings).

**Files to modify/create:**
- Create: `src/base64url.ts` — functions `encodeBase64Url(str)` and `decodeBase64Url(encoded)`
- Create: `tests/unit/base64url.test.ts`

**Instructions for Execution Agent:**

   - Decode URL-safe base64 back to original URL
   - Decode handles URL-safe chars (`-` → `+`, `_` → `/`)
   - Decode handles stripped padding
   - Round-trip: encode then decode returns original string
   - Encode produces web-safe output for strings that would normally have `+`, `/`, `=`
2. **Implement** `encodeBase64Url`: use `btoa()`, then replace `+` → `-`, `/` → `_`, strip trailing `=`.
3. **Implement** `decodeBase64Url`: replace `-` → `+`, `_` → `/`, then `atob()`. Padding is optional (restore before `atob` if needed based on length).
4. Run tests, ensure all pass.
5. Commit with message: `feat: add URL-safe base64 encode/decode utility`

### Task 5: Proxy Worker

**Objective:** Implement the proxy worker that receives internal requests from the router, strips identity headers, generates a fake IP, and forwards to the real upstream URL. This is the core of the IP rotation.

**Files to modify/create:**
- Create: `src/proxy.ts` — `handleProxyRequest(request, env, ctx)` function
- Create: `tests/unit/proxy.test.ts`

**Instructions for Execution Agent:**
1. **Write failing tests** covering:
   - Returns 401 if `X-Internal-Auth` header doesn't match `INTERNAL_AUTH_SECRET`
   - Returns 400 if `X-Target-URL` header is missing
   - Forwards request to `X-Target-URL` using `X-Original-Method`
   - Passes through `Authorization`, `Content-Type`, `Accept` headers from the original request (via `X-Forward-*` forwarded headers or the request itself)
   - Strips all Cloudflare/identity headers — the upstream request must NOT contain `CF-Connecting-IP`, `CF-RAY`, `CF-Visitor`, `CF-IPCountry`, `X-Real-IP`, `Host` (from original)
   - Sets `X-Forwarded-For` to the deterministic fake IP based on PROXY_INDEX + target domain
   - Sets `Host` header to the target domain

   - Response from upstream is returned as-is (status, headers with CORS added, body streamed)
   - Returns 502 if upstream fetch throws an error
2. **Implement** `handleProxyRequest`:

   - Read body as text (pass-through)
   - Extract target domain from URL for fake IP generation
   - Generate fake IP via `generateFakeIp(env.PROXY_INDEX, domain)`
   - Build new headers: copy only `Authorization`, `Content-Type`, `Accept` from forwarded headers; set `Host` to target; set `X-Forwarded-For` to fake IP
   - `fetch(targetUrl, { method, headers, body })`
   - Return upstream response with CORS headers added
3. Run tests, ensure all pass.

### Task 6: Router Worker

**Objective:** Implement the router worker that parses the URL path format `/{PASS}/{PROXY_NUM}/{BASE64_URL}/{...path}`, validates auth, decodes the target URL, selects a proxy by modulo, and forwards via service binding.

**Files to modify/create:**
- Create: `src/router.ts` — `handleRouterRequest(request, env, ctx)` function
- Create: `tests/unit/router.test.ts`

**Instructions for Execution Agent:**
1. **Write failing tests** covering:
   - Returns 403 if PASS segment doesn't match AUTH_KEY
   - Returns 400 if PROXY_NUM is not a valid integer
   - Returns 400 if BASE64_URL segment is invalid base64 or doesn't decode to a valid URL
   - Decodes BASE64_URL correctly using `decodeBase64Url`
   - Constructs real URL: decoded_base + "/" + remaining path segments (e.g., `chat/completions`)
   - Selects proxy: PROXY_NUM % PROXY_COUNT, then uses env binding `PROXY_${index}` (1-based)
   - Forwards request to selected proxy with internal headers: `X-Internal-Auth`, `X-Target-URL`, `X-Original-Method`
   - Passes through client body untouched
   - Returns 502 if resolved proxy binding is undefined
   - CORS preflight (OPTIONS) returns 204 immediately
   - Modulo wraps: PROXY_NUM=100 with 3 proxies → proxy index 100%3=1 → PROXY_2
2. **Implement** `handleRouterRequest`:
   - Handle OPTIONS → return CORS preflight
   - Parse URL path: split `/`, segments[1]=PASS, segments[2]=PROXY_NUM, segments[3]=BASE64_URL, segments[4..]=extra path
   - Validate PASS against `env.AUTH_KEY`
   - Validate PROXY_NUM is integer
   - Decode BASE64_URL, validate it's a real URL
   - Build real URL: decoded + "/" + segments.slice(4).join("/")
   - Select proxy: PROXY_NUM % env.PROXY_COUNT → PROXY_${index} (1-based)
   - Build internal request with headers, forward body
   - Return proxy response
3. Run tests, ensure all pass.
4. Commit with message: `feat: implement router with URL parsing, auth, proxy selection`
